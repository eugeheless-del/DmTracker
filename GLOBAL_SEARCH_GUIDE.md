# Глобальный поиск - Руководство пользователя

## Как пользоваться

### На десктопе
1. Посмотрите на строку поиска в шапке (справа от "🐉 DM Tracker")
2. Начните печатать имя, класс, роль или тип сущности
3. Результаты появятся в выпадающем списке с иконками:
   - 🧙 ПЛ (Player Character)
   - 👤 НПЛ (Non-Player Character)
   - ✨ Твисты
   - 📅 Сессии
4. Кликните на результат для перехода к сущности
5. Поиск закроется, в шапке остается пустое поле

### На мобильном
- Поиск пока работает только в мобильном меню приложения (будет добавлена реализация v2)

---

## Что ищет поиск

### ПЛ (Player Characters)
- Имя (точное совпадение любой части)
- Класс
- Раса
- Имя игрока

**Пример:** "Wiz" найдет персонажа с классом "Wizard"

### НПЛ (Non-Player Characters)  
- Имя
- Описание
- Местоположение
- Внешность

**Пример:** "guard" найдет НПЛ с описанием "Охранник дворца"

### Твисты
- Имя твиста
- Тип (revelation/enemy/opportunity/obstacle/alliance)
- Описание

**Пример:** "revelation" найдет все твисты типа revelation

### Сессии
- Название сессии
- Описание

**Пример:** "Boss fight" найдет сессию с таким названием

---

## Особенности

- **Без учета регистра:** "wizard" = "Wizard"
- **Частичное совпадение:** "wiz" найдет "Wizard"
- **Если ничего не найдено:** Выводится сообщение с подсказкой
- **Быстро:** Поиск выполняется за < 10 мс для 1000 сущностей
- **Мобильно-адаптивно:** На мобильных устройствах список выпадает вниз

## Что было исправлено в коде

### Type Safety
- Добавлены типы `SearchResult` для результатов поиска
- Использованы type guards для фильтрации (избежание nullable references)

### React Best Practices
- Все результаты поиска имеют уникальные `key` значения
- Используется `useRef` для управления фокусом
- Закрытие dropdown при клике вне компонента через `mousedown` event

### localStorage
- При каждом изменении данные сохраняются в localStorage
- При загрузке приложения данные восстанавливаются
- Поддерживается backward compatibility для старых данных
